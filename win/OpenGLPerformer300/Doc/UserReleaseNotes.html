User Release Notes for OpenGL PerformerTM 3.0

This document is intended to answer the most common questions regarding the use of OpenGL Performer 3.0 for Microsoft® Windows NT®, Windows® 2000, and Windows® XP.

For anything not covered here please contact your support provider or performer-feedback@sgi.com.

The following topics are covered in this document:

  • Directory structure/shipped source, data, etc.
  • List of unsupported features
  • File Loaders
  • Known bugs
  • Obtaining license key
  • Reporting problems
 

Supported platforms

OpenGL Performer 3.0 for Microsoft Windows supports Microsoft Windows NT 4.0, Microsoft Windows 2000 and Microsoft Windows XP.  The release has been tested with the latest patch level of each release; Windows NT 4.0 Service Pack 6, Windows 2000 Service Pack 3, and Windows XP Service Pack 1 respectively.

Hardware configurations tested

Our tests were conducted on Silicon Graphics 330 Visual Workstations running both Microsoft Windows 2000 and XP with the latest service packs installed and latest BIOS images.

Environment variables and installation/uninstallation process

During the installation of OpenGL Performer, the following environment variables are registered for both the current user environment and for the system environment:
  • PFROOT
  • PFPATH
These are removed from the system upon uninstallation. Note that the PATH environment variable is also modified such that it contains the two following additional entries:
  • %PFROOT%\Bin
  • %PFROOT%\Lib

Directory structure/shipped source, data, etc

On systems running IRIX® or Linux®, OpenGL Performer libraries are shipped in /usr/lib and all source and data can be found in the /usr/share/Performer directory. On Microsoft Windows, everything is placed relative to the installation directory which also corresponds to the environment variable PFROOT.

The directory structure mapping looks like this:
 

IRIX/Linux Windows
Optimized Libraries /usr/lib[32|64] %PFROOT%/Lib
Debug Libraries /usr/lib[32|64]/Performer/Debug %PFROOT%/Lib/Debug
Sample Source /usr/share/Performer/src %PFROOT%/Src
Sample Data /usr/share/Performer/data %PFROOT%/Data
File Loaders (opt) /usr/lib[32|64]/libpfdb %PFROOT%/Lib/libpfdb
File Loaders (dbg) /usr/lib[32|64]/Performer/Debug/libpfpfb %PFROOT%/Lib/Debug/libpfdb
Binaries (executables) /usr/sbin %PFROOT%/Bin
License file /usr/share/Performer/license.dat %PFROOT%/license.dat

List of currently unsupported features

The list of features not supported on Microsoft Windows is a superset of the list of unsupported features on Linux platforms. In addition, this version of OpenGL Performer for Microsoft Windows supports single-process operation only (PFMP_APPCULLDRAW).

IRIX specific features which are not supported are support for calligraphic light point boards, real-time features, datapools, direct file I/O, and multi-pipe / multi screen capability.

List of file loaders currently ported

Most file loaders have been ported, with the exception of the following:
  • .pfso: Performer shared object
  • .rgb: SGI rgb image loader
  • .ptu: Performer Terrain Utility Format
  • .iv: Open InventorTM
  • .s1k: U.S. Army SIMNET
  • .stm: Simple Terrain Model
During the installation process all model file formats that OpenGL Performer can read will be associated with perfly, if no other association already exists.

Known bugs

  • pop-up menus in perfly don't show up since pup.c in libpfutil is not yet ported. The menu buttons do toggle state if they are clicked on, just the pop-up selectors are unavailable.

Obtaining a license key

From the Start Menu select "SGI OpenGL Performer -> Get HostID Key" and give this information to your salesperson.

Reporting problems to your support provider

When sending a report please be sure to include the following information:

Hardware configuration:

  • processor count/type/speed
  • video card
    • Software configuration:
  • Operating system and service pack level
  • Display depth and resolution
  • Version of video driver used
    • In addition, please run your programs with the following environment variable setup:
  • PFNFYLEVEL = 7
  • _PF_DISPLAY_LOADED_MODULES = 1
    • Also please include the output from running the command 'set' at a DOS prompt. This will help us verify that the following environment variables have been set properly:
  • PFROOT
  • PATH
  • PFPATH
    • You may also want to try setting your PATH to use the debug versions of the libraries. In a DOS shell type the following commands:
  • set PATH=%PFROOT%\Lib\Debug;%PFROOT%\Lib\Debug\libpfdb;%PATH%
